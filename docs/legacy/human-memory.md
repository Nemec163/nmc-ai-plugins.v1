## КОНЦЕПТ ЧЕЛОВЕКО-ПОДОБНОЙ ПАМЯТИ ДЛЯ АИ

## L0 — Working Memory (Runtime Context)

**Назначение:** “сознание” на время текущей задачи.
**Содержимое:** текущая цель, план, загруженные фрагменты памяти, промежуточные выводы.
**Срок жизни:** минуты/часы, не считается “истиной”.
**Кто пишет:** любой агент во время выполнения.
**Кто читает:** агент/оркестратор прямо сейчас.
**Связи:** вытягивает данные из L1–L4, но сам ничего “навечно” не фиксирует.

---

## L1 — Candidates Inbox (Raw Observations)

**Назначение:** буфер “сырого опыта” (как дневной входной ящик мозга).
**Содержимое:** атомарные claims/наблюдения + метаданные:

* `who/where` (источник: агент, канал)
* `when` (timestamp)
* `what` (текст/нормализованное утверждение)
* `context_link` (ссылка на диалог/таск)
* `entities/tags` (о чём)
* `confidence` (насколько уверен экстрактор)

**Срок жизни:** до консолидации (дни/недели), потом архив/удаление по политике.
**Кто пишет:** extractor (или агенты напрямую, но лучше через extractor).
**Кто читает:** Memory Curator, аудит/дебаг.
**Связи:** из L1 рождаются записи в L2–L4. L1 — источник “доказательств” наряду с L2.

---

## L2 — Episodic Timeline (Events, Append-only)

**Назначение:** истина “что произошло” во времени. Это *главный анти-амнезийный слой*.
**Содержимое:** события (events) в форме:

* `event_type`
* `timestamp`
* `entities involved`
* `payload` (структурные поля)
* `evidence` (ссылки на L1, диалоги, документы)
* `confidence`
* `causality/link` (опционально: связи event→event)

**Инвариант:** **append-only**. Ничего не перезаписываем.
**Исправления:** только новым событием (`correction_event`, `clarification_event`).
**Кто пишет:** только Memory Curator; helper subagents могут помогать с extraction/analysis, но publish-authority остается у root-curator.
**Кто читает:** все (через поиск/вьюхи), особенно для “истории/почему”.
**Связи:**

* L3 и L4 *должны* иметь ссылки на L2 как evidence.
* L5 (state) — проекция L2.

---

## L3 — Semantic Knowledge (Stable / Versioned Facts)

**Назначение:** “что я знаю” (обобщённые знания), не обязательно “что сейчас”.
**Содержимое:** факты/навыки/предпочтения и т.п. в виде:

* `subject / predicate / object` (или любая норм. схема)
* `confidence`
* `evidence` (обязательная ссылка на L2/L1)
* `valid_from / valid_to` (опционально, если факт временной по природе)
* `notes/source`

**Правило:** если факт может измениться завтра — либо **State (L5)**, либо **Semantic + валидность**, но не “вечный факт без времени”.
**Кто пишет:** Memory Curator (и только он).
**Кто читает:** все агенты для быстрых ответов/компетенции.
**Связи:**

* Питается из L2 (компиляция паттернов).
* Может ссылаться на L4 (identity) как контекст интерпретации, но не наоборот.

---

## L4 — Identity Layer (Current Identity + Evolution)

**Назначение:** “кто это” — текущая модель личности и устойчивых паттернов.
**Содержимое (минимально):**

* `identity.current` (ценности, стиль мышления, предпочтения общения, цели высокого уровня)
* `identity.updated_at`
* `identity.evidence` (ссылки на L2/L1)
* (опционально) `identity.versions` (если хочешь версионирование прямо тут)

**Правило обновления:** редко и только при пороге доказательств (несколько независимых эпизодов + высокая уверенность).
**Изменения личности:** всегда фиксируются в L2 как события; L4 — “текущая проекция”.
**Кто пишет:** Memory Curator.
**Кто читает:** оркестратор и любые агенты для персонализации/принятия решений.
**Связи:**

* Основан на L2 (эволюция через события).
* Влияет на L0 (как агент думает/говорит), но не подменяет факты.

---

## L5 — State (Current Reality Projection)

**Назначение:** “что подтверждено как текущее” — быстро и однозначно.
**Содержимое:** текущие значения:

* `current_employer`
* `current_city`
* `current_projects`
* `current_focus`
* и т.д. + `updated_at` + `source_event_id`

**Инвариант:** обновляется **только** из L2 (событий).
**Интерпретация:** это `current as of last successful consolidation`, а не speculative runtime-now.
**Кто пишет:** Memory Curator (как часть Apply).
**Кто читает:** все (особенно для оперативных ответов “сейчас/актуально”).
**Связи:**

* Проекция L2.
* Может использоваться как фильтр/контекст для поиска по L2/L3.
* Same-day дельта до следующей консолидации живет только в L0/L1/session history.

---

# Главные связи (коротко)

* **L1 → (Curator) → L2**: кандидаты превращаются в события.
* **L2 → L5**: события обновляют текущую реальность.
* **L2 → L3**: события компилируются в знания.
* **L2 → L4**: события (пакетно) обновляют “текущую личность”.
* **L3/L4/L5 → L0**: всё это загружается в рабочий контекст.
